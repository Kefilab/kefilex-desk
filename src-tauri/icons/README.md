# Icons

Placeholder. Real icons land in Phase 31d (polish + distribution).

For the dev / first build, generate all the platform-specific sizes
from the placeholder square PNG that ships in the repo:

```bash
# From the kefilex-desk root
npx @tauri-apps/cli@latest icon assets/placeholder-icon.png
```

This generates all the platform-specific sizes Tauri's `tauri.conf.json` expects (`32x32.png`, `128x128.png`, `128x128@2x.png`, `icon.ico`, `tray.png`) inside `src-tauri/icons/`.

The placeholder is a simple white K on a stone-900 background, generated by a stdlib-only Python script (no third-party image libs required to reproduce). When you have a polished icon ready, drop a new 512×512 PNG over `assets/placeholder-icon.png` and re-run the generator. The 31d Notion sub-page tracks the polished asset.

Until you've run `tauri icon`, `tauri build` will fail with a missing-icon error. `cargo check` and `cargo build --no-default-features` both work without icons.
